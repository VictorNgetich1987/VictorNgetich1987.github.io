Narrative 1 — Software Design & Engineering

[ expand ]

Artifact Description

The Contact Service application was originally created in CS 320: Software Testing, Automation, and Quality Assurance. It is a Java-based service that manages contact records (add, update, delete) with accompanying JUnit tests. The artifact was created in 2024 and demonstrates foundational object-oriented programming and unit testing skills.

Justification for Inclusion

I selected this artifact because it clearly demonstrates growth in software engineering practice. The original code functioned correctly but lacked clear separation of concerns, had minimal input validation, and contained limited test coverage. The enhancement restructured the service layer, added comprehensive validation logic, and expanded the JUnit test suite to cover edge cases and boundary conditions showcasing my ability to write professional-grade, maintainable Java code.

Reflection on Enhancement Process

The most significant challenge was refactoring the validation logic without breaking existing tests. I learned to write tests first (TDD) and then refactor confidently. Instructor feedback encouraged me to also document my design decisions more clearly, which led me to add inline Javadoc comments throughout. This enhancement partially meets Course Outcomes 3 and 4 applying algorithmic principles and using professional tools in computing practice.

Narrative 2 — Algorithms & Data Structures

[ expand ]

Artifact Description

The Task Management component extends the Contact Service project. It manages a collection of tasks associated with contacts, originally storing them in an ArrayList and performing linear searches for retrieval and updates.

Justification for Inclusion

This artifact demonstrates my understanding of algorithmic trade-offs. By replacing the ArrayList with a HashMap keyed on task ID, I improved average-case lookup performance from O(n) to O(1). I also introduced a comparator-based sorting mechanism for displaying tasks by priority or due date demonstrating practical application of Java's Collections framework and algorithm design principles from CS 260.

Reflection on Enhancement Process

I learned to think critically about the right data structure for a given access pattern. The initial instinct to use a list was functional but inefficient at scale. After profiling the code with large datasets, the performance difference became clear. A key challenge was maintaining data consistency when tasks were updated solved by ensuring atomic map updates. This enhancement meets Course Outcome 3 (algorithmic principles and trade-offs) most directly.

Narrative 3 — Databases

[ expand ]

Artifact Description

The database artifact originates from DAD 220: Introduction to SQL. The original project involved basic CREATE, INSERT, SELECT, and UPDATE operations on a simple flat table structure. The enhancement expanded this into a fully normalized, multi-table relational database integrated with the Contact Service application.

Justification for Inclusion

This artifact demonstrates growth from basic SQL to advanced database design. The enhanced schema applies third normal form (3NF) normalization, uses foreign key constraints for referential integrity, and includes stored procedures and views for common query patterns. It showcases my ability to design databases that are both performant and maintainable skills directly relevant to full-stack development roles.

Reflection on Enhancement Process

Normalization was conceptually straightforward but practically challenging when retrofitting an existing schema especially migrating existing data without loss. I learned to plan schema migrations carefully using transaction blocks. Security was also a consideration: I applied parameterized queries in all application-layer SQL to prevent SQL injection, directly applying knowledge from CS 405. This enhancement meets Course Outcomes 4 and 5 professional tools and security mindset.
